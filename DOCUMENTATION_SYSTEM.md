# Punk_OS Documentation System

This document explains the complete documentation system for Punk_OS, including how users can access installation information through the `man punk_os` command.

## Overview

The Punk_OS project now includes a comprehensive, multi-layered documentation system that provides users with easy access to installation information, usage guides, and quick references.

## Documentation Components

### 1. Man Page System (`man punk_os`)

**File**: `punk_os.1`  
**Installed to**: `/usr/local/man/man1/punk_os.1`  
**Access**: `man punk_os` (from anywhere after installation)

The man page provides:
- Quick reference for all commands
- Installation categories and options
- Version manager usage (nvm, pyenv, rustup, Go)
- Docker and PostgreSQL setup instructions
- CLI tool examples (jq, tree, httpie, flameshot, etc.)
- Troubleshooting guides
- Complete command examples

**Benefits**:
- ✅ Always available offline
- ✅ Standard Linux documentation format
- ✅ Searchable (press `/` while viewing)
- ✅ Professional and familiar to developers
- ✅ Fast access from any directory

### 2. Installation Report Generator

**File**: `check-installed.sh`  
**Output**: `INSTALLED.md` (customizable)  
**Usage**: `./check-installed.sh [output-file]`

This script scans the system and generates a comprehensive report including:
- System information (OS, kernel, architecture)
- All installed package managers and version managers
- Programming languages and runtimes with versions
- Development tools and IDEs
- Command-line tools with descriptions
- Databases and container systems
- Productivity applications
- Installed fonts
- Shell configuration
- Quick reference commands

**Benefits**:
- ✅ See exactly what's on your system
- ✅ Track versions of installed software
- ✅ Generate reports for multiple machines
- ✅ Markdown format (easy to read/share)
- ✅ Can be version controlled

### 3. README Documentation

**File**: `README.md`  
**Purpose**: Comprehensive guide for the entire project

Contains:
- Project overview
- Quick start instructions
- Detailed feature descriptions
- Installation examples
- Usage guides
- Configuration tips
- Troubleshooting

### 4. Quick Start Guide

**File**: `QUICK_START.md`  
**Purpose**: Fast onboarding for new users

Provides:
- How to access all documentation
- First-time setup steps
- Common commands after installation
- Documentation hierarchy
- Pro tips for using the system

### 5. Man Page Development Guide

**File**: `MAN_PAGE_GUIDE.md`  
**Purpose**: Help maintainers update the man page

Covers:
- How to install/update the man page
- Man page format (roff/troff)
- When and how to update
- Troubleshooting
- Contributing guidelines

## Installation Flow

### Automatic Installation (Recommended)
When users run `./setup.sh`, they'll be prompted:
```
Install punk_os man page (allows running 'man punk_os')? (y/n)
```

If they answer 'y':
1. Script copies `punk_os.1` to `/usr/local/man/man1/`
2. Sets correct permissions (644)
3. Updates man database with `mandb`
4. User can immediately run `man punk_os`

### Manual Installation
Users can also install just the man page:
```bash
./install-manpage.sh
```

This script:
- Checks for the source file (`punk_os.1`)
- Determines the correct installation directory
- Copies and sets permissions
- Updates the man database
- Verifies the installation

## For Maintainers

### Updating Documentation

1. **When adding new software to setup.sh**:
   - Update `punk_os.1` with new installation category or example
   - Test: `man ./punk_os.1`
   - Reinstall: `./install-manpage.sh`

2. **When changing command-line options**:
   - Update OPTIONS section in `punk_os.1`
   - Update README.md
   - Update QUICK_START.md

3. **When adding new features**:
   - Update all relevant documentation files
   - Add examples to the man page
   - Test the generated documentation

### Testing Changes

```bash
# View man page before installing
man ./punk_os.1

# Check for roff syntax errors
groff -man -Tascii punk_os.1 | less

# Install and verify
./install-manpage.sh
man punk_os
```

## Troubleshooting Documentation

### Man page not found
```bash
# Reinstall
./install-manpage.sh

# Or manually
sudo mandb
```

### Outdated information
```bash
# Regenerate installation report
./check-installed.sh

# Check man page version
man punk_os | head -n 1
```

### Permission issues
Man page installation requires sudo (normal for system documentation).
