# Man Page Guide for Punk_OS

This guide explains how to use and maintain the Punk_OS man page system.

**Compatibility**: Works on Ubuntu 20.04+, Pop!_OS, Linux Mint, Elementary OS, Zorin OS, and all Ubuntu-based distributions.

## What is a Man Page?

A man (manual) page is the traditional Unix/Linux way of providing documentation. Users can access it by typing `man punk_os` from anywhere in the terminal, on any Ubuntu-based distribution.

## Installation

### Automatic Installation (Recommended)

The man page is automatically installed when you run the setup script:

```bash
./setup.sh
# When prompted "Install punk_os man page", answer 'y'
```

### Manual Installation

If you want to install just the man page:

```bash
./install-manpage.sh
```

Or install it manually:

```bash
# Copy the man page to the system
sudo cp punk_os.1 /usr/local/man/man1/

# Set correct permissions
sudo chmod 644 /usr/local/man/man1/punk_os.1

# Update the man database
sudo mandb
```

## Usage

Once installed, users can access the manual from anywhere:

```bash
# View the full manual
man punk_os

# Search within the manual (press '/' while viewing)
man punk_os
# Then type '/' followed by your search term

# View specific sections
man -P 'less -p "DOCKER USAGE"' punk_os
```

## What's Included in the Man Page

The `punk_os` man page includes:

- **NAME & SYNOPSIS** - Quick overview of commands
- **DESCRIPTION** - What Punk_OS does and distribution compatibility
- **OPTIONS** - Command-line flags for setup.sh and check-installed.sh
- **INSTALLATION CATEGORIES** - All software that can be installed
- **USAGE EXAMPLES** - How to run installations on any Ubuntu-based system
- **VERSION MANAGERS** - Commands for nvm, pyenv, rustup, Go
- **DOCKER USAGE** - Docker and Docker Compose commands
- **POSTGRESQL SETUP** - Database setup instructions
- **CLI TOOLS USAGE** - Examples for jq, tree, httpie, etc.
- **TROUBLESHOOTING** - Common issues and solutions
- **REQUIREMENTS** - Ubuntu 20.04+ and distribution compatibility
- **FILES** - Important configuration files
- **ENVIRONMENT VARIABLES** - What gets set
- **EXAMPLES** - Complete workflow examples

## Updating the Man Page

### When to Update

Update the man page when you:
- Add new software to setup.sh
- Change command-line options
- Add new features
- Update usage instructions

### How to Update

1. Edit the `punk_os.1` file using the roff format
2. Test your changes locally:
   ```bash
   # View your changes without installing
   man ./punk_os.1
   ```

3. Reinstall the man page:
   ```bash
   ./install-manpage.sh
   ```

4. Verify the changes:
   ```bash
   man punk_os
   ```

## Man Page Format (Roff)

The man page uses roff/troff markup. Here are common elements:

```roff
.TH TITLE 1 "DATE" "VERSION" "MANUAL_TITLE"    # Header
.SH SECTION_NAME                                # Major section
.SS Subsection Name                             # Subsection
.TP                                             # Tagged paragraph
.B bold_text                                    # Bold text
.I italic_text                                  # Italic text
.BR bold normal                                 # Bold then normal
.PP                                             # New paragraph
.nf / .fi                                       # No-fill / fill (for code blocks)
.RS / .RE                                       # Indent / outdent
.IP \(bu 2                                      # Bullet point
```

### Example Addition

To add a new tool to the man page:

```roff
.SS New Tool Name
Description of what it does:
.nf
command --example          # What it does
command --another-flag     # More examples
.fi
```

## Uninstalling the Man Page

```bash

sudo rm /usr/local/man/man1/punk_os.1

# Update the man database
sudo mandb
```

## Tips for Users

### Quick Navigation

While viewing the man page:

- **Space** - Page down
- **b** - Page up
- **/** - Search forward
- **?** - Search backward
- **n** - Next search result
- **N** - Previous search result
- **q** - Quit
- **h** - Help

### Print to File

```bash
# Convert man page to text file
man punk_os | col -b > punk_os_manual.txt

# Convert to PDF (requires ps2pdf)
man -t punk_os | ps2pdf - punk_os_manual.pdf
```

### Search Without Opening

```bash
# Search for a term in the man page
man -K "docker" punk_os
```

## Troubleshooting

### "No manual entry for punk_os"

**Solution 1**: Update man database
```bash
sudo mandb
```

**Solution 2**: Check if file exists
```bash
ls -la /usr/local/man/man1/punk_os.1
ls -la /usr/share/man/man1/punk_os.1
```

**Solution 3**: Reinstall
```bash
./install-manpage.sh
```

### Man page shows old content

```bash
# Clear man page cache and rebuild
sudo rm -rf /var/cache/man/*
sudo mandb
```

### Permission denied

The installation requires sudo because man pages are installed to system directories. This is normal and expected.
